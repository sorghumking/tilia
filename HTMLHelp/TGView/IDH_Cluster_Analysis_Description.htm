Cluster Analysis

TGView can perform a Constrained Incremental Sum of Squares (CONISS) cluster analysis and display the results as a dendrogram on the diagram. The analysis is carried out from and plotting options are designated in the Cluster Analysis Dialog Box.

 

For more technical information about CONISS and details on analysis and plotting options, see Grimm (1987). This paper describes the original FORTRAN program that ran on a mainframe computer. The description of the algorithm and program options are still valid, although the instructions for running the FORTRAN program and producing graphics are obsolete. The current version of CONISS has been converted to C++ and runs under Windows.

 

Please Note: CONISS includes every variable and sample in the spreadsheet. Therefore, the spreadsheet must not include chronologies, depths, sums, spike counts, sample volumes, and other such variables. The spreadsheet used for a diagram is usually not appropriate for CONISS.  Also, if the data are percentages (or proportions), the values for each sample must add to 100% (or 1.0 for proportions) for the implied dissimilarity coefficient to be valid. It is not valid to have percentages based on different sums. With pollen, for example, aquatics are normally removed from the analysis. If they are included in the cluster analysis, they must be included in the sum with upland types, and their percentages must be based on the same sum as the upland types. If certain variables are not to be used in the analysis, percentages should be calculated after these variables are removed, not before. In general, it is safer to use count data and to have CONISS calculate the percentages. The analysis will usually run with invalid data, but the "garbage in, garbage out" dictum holds true.
