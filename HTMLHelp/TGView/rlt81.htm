Run CONISS

Click in the Cluster Analysis Dialog Box to run CONISS, the cluster analysis program. Clicking this button will activate the Constrained Incremental Sum of Squares Dialog Box.

 

 

 

 

Click File to load a .til file or to exit the analysis. The results are saved to a .dgx file, which TGView can read to plot a dendrogram. Be default, this file will have the same name as the input .til file, but will be given the .dgx extension. The .dgx file name can be changed by typing another name or by clicking to activate the Open File Dialog Box.

 

The Type of Data can be Counts, Percentages, or Proportions. For pollen counts,  Convert to proportions is normally checked. However, the spreadsheet could include non-frequency data that should not be converted to proportions.

 

For Type of Analysis, choose Stratigraphically constrained or Unconstrained. Stratigraphically constrained is normally selected for stratigraphic data. This method forces the analysis to consider only stratigraphically adjacent samples or clusters during the agglomerative process. Unconstrained is typically chosen for surface samples that have no prior stratigraphic order.

 

Four possible data transformations with implied dissimilarity coefficients are provided. The default is Square root transformation (Edwards & Cavalli Sforza's chord distance). For pollen data, this choice is normally the best. This chord distance is the one that appears commonly in the pollen literature. For other kinds of data, one of the other choices may be appropriate. Please see Grimm (1987) and references herein for more information.

Please Note: CONISS includes every variable and sample in the spreadsheet. Therefore, the spreadsheet must not include chronologies, depths, sums, spike counts, sample volumes, and other such variables. Also, if the data are percentages (or proportions), the values for each sample must add to 100% (or 1.0 for proportions) for the implied dissimilarity coefficient to be valid. It is not valid to have percentages based on different sums. With pollen, for example, aquatics are normally removed from the analysis. If they are included in the cluster analysis, they must be included in the sum with upland types, and their percentages must be based on the same sum as the upland types. If certain variables are not to be used in the analysis, percentages should be calculated after these variables are removed, not before. In general, it is safer to use count data and to have CONISS calculate the percentages. Typically, the analysis will still run with invalid data, but the "garbage in, garbage out" dictum holds true.
